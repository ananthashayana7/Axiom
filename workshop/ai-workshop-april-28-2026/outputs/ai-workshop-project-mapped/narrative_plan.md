# AI Workshop Project Mapping Deck Plan

## Audience
Internal AI workshop attendees who need to see practical proof of AI, ML, full-stack software engineering, API integration, data pipelines, security controls, and delivery maturity.

## Objective
Extend the edited workshop presentation with a project mapping section that connects the abstract workshop themes to concrete work from Axiom, Mastiff, and OEE Box. The personal project is intentionally excluded.

## Narrative Arc
The existing slides introduce the workshop story, capability pillars, reference architecture, demos, software stack, governance, and roadmap. The new appendix closes the gap between "what we can talk about" and "what we have actually built" by mapping each pillar to implemented project evidence.

## Added Slide List
1. Project Mapping Overview: cross-project evidence matrix across AI pillars.
2. Axiom: Procurement Intelligence Proof: business AI, agents, APIs, queues, cloud stack, tests.
3. Mastiff: Conversational Data Analyst Proof: upload-to-insight loop, sandboxed Python, charts, exports, enterprise connectors.
4. OEE Box: Edge AI + Industrial Intelligence Proof: machine telemetry, RAG, trust scoring, RL shadow mode, gatekeeper, real-time dashboard.
5. Slide-by-Slide Talking Map: what to say for each workshop theme and which project proves it.
6. Live Demo Run Sheet: practical demo path and closing proof statement.

## Visual System
Use the same workshop visual language: clean off-white background, dark ink text, compact top navigation, colored project cards, pill labels, architecture flows, and dense but readable technical proof points.

## Editability Plan
All added content is editable PowerPoint text and native shapes. No screenshot-only slides, no manual hyperlink XML, and no unsupported interactive patching.
